# Результаты тестирования производительности (Задание 6)

В данном отчете представлены результаты нагрузочного тестирования двух микросервисов: на языке Go (фреймворк Gin) и на языке Python (фреймворк FastAPI). Тестирование проводилось с помощью утилиты Apache Benchmark (ab).

## Окружение и условия
- **Инструмент:** ApacheBench Version 2.3
- **Тестовый стенд:** localhost (127.0.0.1)
- **Go (Gin):** Порт 8080, эндпоинт `/target`, нагрузка 1000 запросов (concurrency 20)
- **Python (FastAPI):** Порт 8000, эндпоинт `/ping`, нагрузка 500 запросов (concurrency 10)

| **Requests per second (RPS)** | 4138.68 [#/sec] | 968.01 [#/sec] |
| **Time per request (mean)** | 4.832 ms | 10.330 ms |

## Анализ и выводы
1. **Производительность:** Сервис на Go (Gin) продемонстрировал пропускную способность в **4.27 раз выше**, чем сервис на Python (FastAPI).
2. **Задержки (Latency):** Среднее время ответа у Go составляет ~4.8 мс против ~10.3 мс у Python, что делает Go-сервис более отзывчивым под нагрузкой.
3. **Стабильность:** 99% всех запросов в Go уложились в 8 мс, в то время как у Python этот показатель составил 15 мс.

**Итоговый вердикт:** Тестирование подтверждает преимущество компилируемого языка Go в задачах с высокой сетевой нагрузкой. Фреймворк Gin эффективнее утилизирует ресурсы процессора, обеспечивая более высокую плотность обработки запросов на одном и том же оборудовании.
